## Overview

Webhooks enable your system to automatically receive event notifications from Trackpilots.  
When a selected event occurs, Trackpilots sends a **POST request** to your configured webhook URL with the event payload.

You can create and manage webhooks from the **Developer Tools** section in the Trackpilots dashboard.

---

## Create a Webhook

Follow the steps below to create a new webhook:

1. Go to **Developer Tools** in your Trackpilots dashboard.
2. Select **Webhooks** from the menu.  
   üëâ Direct link:  
   `https://app.trackpilots.com/developer-tools/webhooks`
3. Click the **Create Webhook** button located at the **top-right corner**.
4. Enter the following details:
   - **Webhook Name** ‚Äì A friendly name to identify the webhook.
   - **Webhook URL** ‚Äì The endpoint where event notifications will be sent.  
     Example:
     ```text
     https://trackpilots.com/path
     ```
5. Select the events you want to receive notifications for.
6. Click **Create**.

üéâ Your webhook will be created successfully.

## Event Selection

You can choose specific events or enable all events at once.

### Select All Events

- Receive notifications for **all supported events**.

### Desktop Events

The following desktop-related events are currently supported:

- `desktop.screenshot_tracking.captured`
- `desktop.app_tracking.captured`
- `desktop.activity_tracking.captured`

Each selected event will trigger a webhook notification when it occurs.

---

## Webhook Behavior

- Trackpilots sends a **POST request** to the configured webhook URL.
- The request contains event details in **JSON format**.
- Your endpoint should respond with a **2xx HTTP status code** to acknowledge successful delivery.
